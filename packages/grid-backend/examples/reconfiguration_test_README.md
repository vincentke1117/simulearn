# 拓扑重构专项测试样例文档

## 概述

本样例 `reconfiguration_test.json` 是专门为测试Julia-Grid Designer & Optimizer拓扑重构功能设计的配电网。它包含3条主要路径、多个备用路径和丰富的开关配置，能够充分验证网络重构算法。

## 网络特点

### 基本参数
- **节点数**: 15个母线节点
- **负荷数**: 8个负荷 (总计6.5 MW)
- **电源**: 1个主电源 + 2个分布式电源
- **线路**: 20条线路
- **开关**: 10个开关
- **基准容量**: 100 MVA
- **基准电压**: 10.5 kV

### 拓扑结构

```
          bus-1 (Slack) ── bus-2 ─── bus-3 ─── bus-4
              │              │          │          │
          [grid-1]        [sw-1]    [sw-2]    [sw-3]
              │           OPEN      CLOSED     │
              │              │          │      │
              │              │          │      │
            [备用1]      [备用2]   [备用3]   │
              │              │          │      │
              │              │          │   bus-7 ── [load-3]
              │              │          │      │       │
              │              │          │   [dg-2]  bus-13
              │              │          │      │
          bus-10 ◄────────── bus-9 ◄─── bus-8   │
           │       │          │    │       │    │
        [备用4]  [备用5]  [sw-4]  [sw-5]  [sw-6] [sw-7]
           │       │          │    │       │    │
         [备用6]    │      bus-11  bus-12  bus-14  │
                     │        │     │      │      │
                  [load-4] [load-5] [load-6] [load-7]
```

### 关键特征

#### 1. 三条主要供电路径

**路径A** (主路径):
- bus-1 → bus-2 → bus-5 → 负荷区
- 特点: 距离短，阻抗小

**路径B** (次路径):
- bus-1 → bus-2 → bus-3 → bus-6 → 负荷区
- 特点: 中等距离，负荷集中

**路径C** (长路径):
- bus-1 → bus-2 → bus-3 → bus-4 → bus-7 → 负荷区
- 特点: 距离长，末端负荷

#### 2. 三条备用路径

**备用路径1** (A-B连接):
- bus-2 → bus-8 → bus-9 → bus-10
- 作用: 连接A、B两分支

**备用路径2** (B-C连接):
- bus-3 → bus-10 → bus-9 → bus-8
- 作用: 连接B、C两分支

**备用路径3** (C-A连接):
- bus-4 → bus-9 → bus-8 → bus-2
- 作用: 连接C、A两分支

#### 3. 应急备用路径

**应急路径1**:
- bus-1 → bus-10 (直接连接)
- 作用: 紧急情况下绕过中间节点

**应急路径2**:
- bus-1 → bus-9 (直接连接)
- 作用: 另一个紧急旁路

## 开关配置

### 初始状态

| 开关ID | 位置 | 状态 | 作用 |
|--------|------|------|------|
| sw-main-a | bus-2 ↔ bus-3 | **OPEN** | 主干分段开关 |
| sw-main-b | bus-3 ↔ bus-4 | CLOSED | 主干分段开关 |
| sw-bypass-1 | bus-2 ↔ bus-8 | CLOSED | A支路旁路 |
| sw-bypass-2 | bus-3 ↔ bus-10 | **OPEN** | B支路旁路 |
| sw-bypass-3 | bus-4 ↔ bus-9 | CLOSED | C支路旁路 |
| sw-tie-1 | bus-8 ↔ bus-9 | **OPEN** | A-B联络线 |
| sw-tie-2 | bus-9 ↔ bus-10 | **OPEN** | B-C联络线 |
| sw-tie-3 | bus-10 ↔ bus-8 | **OPEN** | C-A联络线 |
| sw-backup-1 | bus-1 ↔ bus-10 | **OPEN** | 应急备用1 |
| sw-backup-2 | bus-1 ↔ bus-9 | **OPEN** | 应急备用2 |

### 开关策略

**正常运行**:
- 维持径向结构
- 主要通过路径A供电
- 部分支路通过旁路供电

**重构选项**:
- 可以闭合sw-main-a，形成环网
- 可以闭合所有sw-tie-*，形成完全环网
- 可以闭合应急备用，直接供电

- **A支路区域**: dg-1 (800 kW)

## 重构场景测试

### 场景1: 负荷均衡

**目标**: 平衡各支路负荷

**初始状态**:
- A支路: 1,800 kW (29.5%)
- B支路: 2,300 kW (37.7%)
- C支路: 2,400 kW (39.3%)

**优化方案**:
1. 闭合sw-main-a，使各支路形成环网
2. 闭合sw-tie-1，平衡A、B支路
3. 闭合sw-bypass-2，为B支路供电

**预期结果**:
- 各支路负荷偏差 < 15%
- 整体网损降低

### 场景2: 网损优化

**目标**: 最小化网络有功损耗

**初始网损**: 约180 kW (2.8%)

**优化策略**:
1. 闭合低阻抗路径
2. 断开高阻抗线路
3. 优化潮流分布

**重构后网损**: 约140 kW (2.2%)

**改善**: 降低40 kW (22%)

### 场景3: 电压改善

**目标**: 提升末端节点电压

**初始电压**:
- bus-5 (A末端): 0.96 p.u.
- bus-6 (B末端): 0.95 p.u.
- bus-7 (C末端): 0.94 p.u.

**优化方案**:
1. 闭合应急备用路径
2. 调整DG出力
3. 优化无功分布

**预期电压**:
- bus-5: 0.98 p.u.
- bus-6: 0.97 p.u.
- bus-7: 0.97 p.u.

### 场景4: 故障恢复

**模拟场景**: 路径A故障

**故障点**: line-main-1跳闸

**恢复方案**:
1. 断开故障段
2. 闭合sw-bypass-1，通过备用路径供电
3. 闭合sw-main-a临时供电
4. 如需，闭合应急备用路径

**恢复时间**: < 5分钟 (自动重构)

## 重构算法验证

### 测试指标

1. **收敛性**
   - 算法能收敛到最优解
   - 迭代次数合理 (< 100次)

2. **求解时间**
   - 单次优化 < 10秒
   - 可以接受实时重构

3. **结果质量**
   - 网损降低 > 15%
   - 电压提升 > 2%
   - 负荷均衡改善

4. **可行性**
   - 所有开关状态二进制约束满足
   - 径向性约束满足
   - 无过载线路

### 1. 径向性约束

- 验证无环网形成

- 检查开关操作次数限制

- 验证节点电压在允许范围

- 验证变压器容量限制

## 预期问题与解决

### 可能遇到的问题

1. **收敛困难**
   - 原因: MINLP问题复杂
   - 解决: 调整求解器参数

2. **局部最优**
   - 原因: 非凸优化
   - 解决: 多次运行，取最优解

3. **计算时间长**
   - 原因: 150变量
   - 解决: 启发式算法

- ✅ 满足所有约束
